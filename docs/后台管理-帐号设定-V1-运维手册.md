# AmyClaw 后台管理程序 — 帐号设定（V1 运维手册）

本文档面向**运维人员**，说明管理后台的访问凭证、系统用户密码、环境变量、存储路径及故障排查。

---

## 一、帐号体系概览

| 类型 | 用途 | 设置/存储位置 | 校验方式 |
|------|------|----------------|----------|
| **管理后台登录凭证** | 访问 http://&lt;本机IP&gt;:8080 设置页、调用需鉴权 API | Token，存于 `data/token.txt` 与 `openclaw.json`（gateway.auth.token） | Cookie `openclaw_management_token` 或请求头 `Authorization: Bearer <token>` |
| **系统管理员** | 本机 root 登录（SSH/控制台） | 由「系统与用户密码」通过 `chpasswd` 写入 /etc/shadow | 系统登录 |
| **标准用户** | 本机普通用户登录（默认用户名 `amyclaw`） | 同上 | 系统登录 |

管理后台的「Token」与系统用户密码**相互独立**：Token 仅用于访问 8080 管理页与 API；root/amyclaw 密码仅用于本机系统登录。

---

## 二、管理后台 Token

### 2.1 含义与作用

- **Token** 是访问管理后台（端口 8080）的**唯一登录凭证**。
- 与 Gateway（18789）鉴权共用同一 Token：保存时写入 `openclaw.json` 的 `gateway.auth.token` 与 `gateway.remote.token`，并同步到 `data/token.txt`。
- 未设置 Token 时，打开 8080 会看到「设置 Token」界面；已设置则需输入该 Token 才能进入设置页或调用需鉴权接口。

### 2.2 存储位置

| 部署方式 | Token 文件路径 |
|----------|----------------|
| 从仓库运行（如开发） | `openclaw-management/data/token.txt`（相对于 server.js 所在目录） |
| 量产部署到 /opt | `/opt/openclaw-management/data/token.txt` |

配置中的 Token 同时写入 `OPENCLAW_STATE_DIR/openclaw.json`（`config.gateway.auth.token`、`config.gateway.remote.token`）。  
**OPENCLAW_STATE_DIR** 默认 `~/.openclaw`（以运行进程的用户为基准）；量产若使用自定义目录，需与 Gateway 一致（见环境变量一节）。

### 2.3 首次设置与登录流程

1. 浏览器访问 `http://<本机IP>:8080`。
2. 若设备**尚未设置** Token：  
   - 点击「自动生成」或手动输入 Token，再点击「确认并保存」。  
   - 服务端执行：写入 `openclaw.json`（gateway.auth.token / gateway.remote.token）、写入 `data/token.txt`，并可选写入 `.env` 中的 `OPENCLAW_GATEWAY_TOKEN`（视实现而定）。  
3. 若设备**已设置** Token：  
   - 在输入框中输入当前 Token，点击「确认并保存」完成验证，随后进入设置页（Cookie 会保存，下次可免输）。

### 2.4 校验逻辑（供排查用）

- **getAuthToken(req)**：优先从 `Authorization: Bearer <token>` 取，否则从 Cookie `openclaw_management_token` 取。
- **requireAuth**：  
  - 用请求中的 token 与 `data/token.txt` 中内容比较；  
  - 若不一致，再与 `openclaw.json` / `.env` 中的 Gateway Token 比较；若一致则顺带写入 `token.txt` 并放行。  
- 因此：**只要 Gateway Token 与 8080 使用的 Token 一致**，用任一方式保存一次即可两边通用。

### 2.5 修改与清除 Token

- **修改**：登录后可在「当前 Token」区块点击「更改 Token」，输入新 Token 并确认保存；会同时更新 `openclaw.json` 与 `data/token.txt`。
- **清除**：  
  - API：**DELETE /api/token**（需鉴权）：删除 `data/token.txt` 并移除 `openclaw.json` 中的 gateway token。  
  - 或运维手动删除 `data/token.txt` 并编辑 `openclaw.json` 去掉 `gateway.auth.token`、`gateway.remote.token`。  
- 清除后再次访问 8080 会回到「未设置 Token」的初始状态，需重新设置或输入 Token。

## 三、系统与用户密码（root / 标准用户）

### 3.1 说明

- 在管理页「系统与用户密码」区块可修改本机 **root** 与**标准用户**的登录密码。
- **标准用户名**由环境变量 **STANDARD_USER_NAME** 指定，默认 **amyclaw**；前端通过 **GET /api/system/users** 获取并展示（返回 `root`、`standardUser`）。
- 修改密码由服务端 **POST /api/system/change-password**（需鉴权）调用系统 `chpasswd` 完成；**管理进程必须以 root 运行**才能成功执行。

### 3.2 密码规则（服务端校验）

- 不少于 **8 位**。
- 必须同时包含：**小写字母**、**大写字母**、**数字**。  
- 对应实现：`validateSystemPassword()`，不满足时返回 400 及具体提示（如「需包含大写字母」）。

### 3.3 API 说明

- **GET /api/system/users**  
  - 无需鉴权（或按当前实现）。  
  - 返回：`{ root: "root", standardUser: "<STANDARD_USER_NAME>" }`。

- **POST /api/system/change-password**  
  - 需鉴权（Cookie 或 Bearer Token）。  
  - Body：`{ user: "root" | "<standardUser>", password: string }`。  
  - `user` 仅允许 `"root"` 或当前 `STANDARD_USER_NAME`（如 `"amyclaw"`），否则 400。  
  - 密码通过 `validateSystemPassword` 校验后，用 `chpasswd` 写回系统。

### 3.4 与「清除所有帐号」的区别

- **一键清除所有帐号**（如 **POST /api/clear-all-accounts**）会清除：Gateway Token、飞书、大模型/向量等 API Key、`data/token.txt` 以及 `.env` 中的敏感键。  
- **不会**清除或修改 root / 标准用户的系统密码；系统密码仅通过「系统与用户密码」或 **POST /api/system/change-password** 修改。

## 四、环境变量（运维相关）

| 变量 | 说明 | 默认值 |
|------|------|--------|
| **OPENCLAW_STATE_DIR** | 状态目录，存放 `openclaw.json`、`.env`；**必须与 Gateway 一致** | `~/.openclaw` |
| **STANDARD_USER_NAME** | 标准用户（普通用户）名，用于密码修改展示与 API 校验 | `amyclaw` |
| **MANAGEMENT_PORT** | 管理服务监听端口 | `8080` |
| **OPENCLAW_GATEWAY_URL** | Gateway 地址，用于管理端探测/调用 | `http://127.0.0.1:18789` |
| **UPGRADE_SERVICE_URL** / **REMOTE_MANAGEMENT_URL** 等 | 升级、远程管理链接等 | 见 README |

- 量产部署时若状态目录不在默认 `~/.openclaw`，启动管理服务与 Gateway 时都需设置相同的 **OPENCLAW_STATE_DIR**。
- 修改 **STANDARD_USER_NAME** 后需重启管理服务，前端与 **POST /api/system/change-password** 的 `user` 允许值会随之变化。

---

## 五、数据与权限建议

- **data/token.txt**：建议仅 root 或运行管理进程的用户可读（如 `chmod 600`），避免 Token 泄露。
- **OPENCLAW_STATE_DIR** 下的 `openclaw.json`、`.env` 含密钥与 Token 引用，同样建议限制为运行用户可读。
- 管理进程需 **root** 才能成功调用 `chpasswd`；若以普通用户运行，修改系统密码会失败（500）。

## 六、故障排查

| 现象 | 可能原因 | 处理建议 |
|------|----------|----------|
| 访问 8080 一直提示未授权 / 401 | Token 未设置、错误或未带在请求中 | 确认 `data/token.txt` 存在且内容正确；用正确 Token 在页面上「确认并保存」，或请求头带 `Authorization: Bearer <token>` |
| 忘记管理后台 Token | 无法通过界面找回 | 用 root 查看 `data/token.txt`，或重新设置：删除该文件并清空 `openclaw.json` 中 gateway token 后刷新 8080，重新生成/填写 Token |
| 修改 root 或用户密码报 500 | 管理进程未以 root 运行；或 `chpasswd` 不可用 | 以 root 启动管理服务（如 systemd 的 User=root）；检查 `chpasswd` 是否在 PATH |
| 修改密码报 400「需包含…」 | 新密码不满足 8 位+大小写+数字 | 按提示调整密码后重试 |
| 保存配置后 Gateway 未生效 | 管理端与 Gateway 使用的 **OPENCLAW_STATE_DIR** 不一致 | 两边设为同一目录并重启 Gateway；必要时检查 `GET /api/status` 返回的 configPath/envPath |
| 标准用户名显示不对 | 与预期不符 | 检查 **STANDARD_USER_NAME** 环境变量并重启管理服务；GET /api/system/users 会返回当前 standardUser |

## 七、相关文件与接口速查

- **Token 文件**：`<管理端安装目录>/data/token.txt`（如 `/opt/openclaw-management/data/token.txt`）。
- **配置与密钥**：`OPENCLAW_STATE_DIR/openclaw.json`、`OPENCLAW_STATE_DIR/.env`。
- **关键 API**：  
  - **POST /api/set-token**：设置/覆盖 Token（无需鉴权）。  
  - **GET /api/status**：是否已设置 Token、Gateway 是否可达、stateDir/configPath/envPath。  
  - **GET /api/system/users**：root 与 standardUser 名称。  
  - **POST /api/system/change-password**：修改 root 或标准用户密码（需鉴权）。  
  - **DELETE /api/token**：清除 Token（需鉴权）。

以上为后台管理程序帐号设定的运维侧完整说明，便于部署、排错与权限管理。
