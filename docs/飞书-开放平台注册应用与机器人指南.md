# 飞书：账号、开放平台应用与机器人 — 完整流程说明

> **定位**：从零开始，在飞书侧完成 **账号 → 组织/租户 → 自建应用 → 机器人能力 → 权限与事件 → 获取 App ID / Secret**，供 AmyClaw 管理页填写。  
> **说明**：飞书开放平台界面会随版本调整，**以飞书官网当前页面为准**；本文给出**必做步骤清单**与 AmyClaw 侧对应关系。

---

## 一、准备：你需要有什么？

| 项目 | 说明 |
|------|------|
| **飞书账号** | 个人手机号或邮箱注册即可；企业使用需企业管理员配合。 |
| **自建应用权限** | 在目标企业中，需有权限 **创建自建应用**（一般企业管理员或应用管理员）。 |
| **网络** | AmyClaw 设备能访问飞书开放平台与飞书 API（出网）；若用 **Webhook** 模式，还需公网可达 URL（见文末「与 AmyClaw 连接模式」）。 |

---

## 二、注册飞书账号（个人/企业）

1. 打开 **[飞书官网](https://www.feishu.cn/)** 或 App，选择 **注册**。  
2. 使用 **手机号** 或 **邮箱** 完成验证。  
3. 若为企业使用：由企业管理员邀请你加入企业，或创建企业（按飞书向导操作）。  
4. 登录 **飞书客户端**（桌面或手机），确认能进入工作台。

> **注意**：后续「开放平台」里创建的应用，属于**某个企业/租户**；机器人只能在该租户可见范围内使用（需把机器人加到对应群或允许单聊）。

---

## 三、进入飞书开放平台

1. 浏览器打开 **[飞书开放平台](https://open.feishu.cn/)**。  
2. 使用**同一飞书账号**登录。  
3. 若提示选择 **企业/租户**，选择要部署机器人的那个。

---

## 四、创建企业自建应用

1. 在开放平台首页进入 **「应用管理」** 或 **「创建企业自建应用」**（文案可能为「创建应用」）。  
2. 填写 **应用名称**、**描述**（可随意填写便于识别，如「AmyClaw 助手」）。  
3. 创建完成后进入该应用的 **详情页**。  
4. 记录以下信息（AmyClaw 管理页会用到）：  
   - **App ID**（`cli_` 开头）  
   - **App Secret**（在「凭证与基础信息」中生成/查看，**勿泄露**）

---

## 五、启用「机器人」能力

1. 在应用详情左侧找到 **「应用能力」** / **「机器人」**（名称以页面为准）。  
2. **启用机器人**，保存。  
3. 部分版本需设置 **机器人名称、头像、描述**，按提示保存。

---

## 六、配置权限（Scope）

机器人要 **收消息、发消息**，至少需 **即时通讯（IM）** 相关权限。在 **「权限管理」** 中：

1. 搜索并开通与 **消息** 相关的权限，常见包括（名称以开放平台为准）：  
   - 获取与发送单聊、群组消息  
   - 读取用户发给机器人的消息（**接收消息**）  
2. **保存**后，若提示 **需要管理员审核**，在企业内提交审核或由管理员通过。

> **提示**：若权限不足，后续会出现 **发不出消息** 或 **收不到事件**；以开放平台 **权限列表** 与 **错误码** 为准排查。

---

## 七、事件订阅：接收消息（关键）

飞书把「用户发消息给机器人」通过 **事件** 推送给你的应用。AmyClaw 默认常用 **长连接**（无需公网域名）。

### 方式 A：长连接（推荐用于内网设备）

1. 在应用内打开 **「事件与回调」** / **「事件订阅」**。  
2. **订阅方式** 选择 **「使用长连接接收事件」**（或等价文案：**不**使用「请求地址 URL」作为唯一方式）。  
3. 在 **「订阅事件」** 中添加 **即时消息** 类事件，至少包含：  
   - **`im.message.receive_v1`**（接收消息）  
4. **保存**。  

> AmyClaw 网关会在服务器上 **主动连接飞书长连接**；设备需能访问飞书服务器出网。

### 方式 B：Webhook（需公网 URL）

若无法使用长连接或官方要求 URL 回调，需配置 **请求地址**、**Encrypt Key**、**Verification Token** 等，并把本机端口暴露到公网。详见 **`feishu-webhook-配置步骤.md`**（与本文并列，不重复）。

---

## 八、发布与可用范围

1. 在 **「版本管理与发布」** 中创建版本并 **发布**（或 **可用范围** 设为 **全员/指定部门**，按企业策略）。  
2. 未发布或可用范围不包含目标用户时，用户可能 **搜不到机器人** 或 **无法添加**。

---

## 九、让用户/群能用到机器人

1. **单聊**：用户在飞书里 **搜索机器人名称** 或从应用页 **添加机器人** 为联系人，再发消息测试。  
2. **群聊**：进入目标群 → **设置** → **群机器人** / **添加机器人** → 选择你的应用机器人。  
3. 若群策略要求 **@机器人**，请在 AmyClaw 飞书配置中确认 **`requireMention`** 等项（见管理页或 `openclaw.json`）。

---

## 十、把凭证填到 AmyClaw

1. 在设备上打开 **`http://<设备IP>:8080`**，登录管理页（Token）。  
2. 在飞书配置中填写 **App ID**、**App Secret**，保存。  
3. 确保 **`plugins.entries.feishu.enabled`** 与 **`channels.feishu.enabled`** 均为启用（若使用较新版本，保存时会自动对齐；否则见 **`热加载与无回复排查.md`**）。  
4. 保存后观察网关日志是否出现 **`starting feishu`** / **`WebSocket client started`**（需运维权限）。

---

## 十一、与 AmyClaw 连接模式对照

| 飞书侧 | AmyClaw 侧（概念） |
|--------|---------------------|
| 长连接接收事件 | `connectionMode` 为 websocket（默认常见） |
| 请求地址 URL | `connectionMode: "webhook"` + 端口/路径 + Token/Encrypt Key |

---

## 十二、常见问题

| 问题 | 可能原因 |
|------|----------|
| 开放平台搜不到「长连接」 | 应用类型非企业自建、或账号权限不足；换企业管理员创建应用。 |
| 机器人已加群但无回复 | 未订阅 `im.message.receive_v1`、权限未生效、或 AmyClaw 侧插件未启用。 |
| 仅群内需 @ 才回复 | `requireMention` 为 true；可在配置中改为 false 并保存（以产品说明为准）。 |

## 十四、多实例运维提醒（必读）

**任何时候**都**不要**在多台 AmyClaw 设备或多套独立 OpenClaw 环境上，**共用同一飞书自建应用**（同一 **App ID + App Secret**）**并行调试**。否则易出现长连接/事件路由混乱、消息重复或漏收、排错结论不可信。  

**建议**：每台设备或每套环境使用**独立飞书应用**（至少独立测试应用）；详见 **`AmyClaw-量产与运维-v2.0.7-release.md`** 第 **5.4** 节。
