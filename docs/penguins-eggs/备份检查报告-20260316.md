## 三、根因与建议

### 3.1 为何 /usr、/var 为空

- `snapshot_dir` 设为 **/mnt/backup**，本机该路径为 **NFS 挂载**。
- eggs 对 `/usr`、`/var` 使用 **overlay 挂载**，将 lowerdir 放在 `snapshot_dir` 下（如 `/mnt/backup/.overlay/lowerdir/usr`）。
- overlay 在 NFS 上常不可用，导致挂载失败，后续 mksquashfs 只打包了空的 `liveroot/usr`、`liveroot/var`。

**建议**：将 **snapshot_dir 改到本地盘**（如 `/var/tmp/eggs` 或 `/home/eggs`），打包完成后再把生成的 ISO **拷贝到 /mnt/backup**。

### 3.2 为何未包含 /mnt/disk/amyclaw

- eggs 对 `/mnt` 的处理是：**只创建空目录** `liveroot/mnt`，不做 bind mount 也不做 rsync/cp。
- 因此无论排除列表如何，`/mnt/disk/amyclaw` 都不会进入镜像。

**建议（二选一或组合）**：

1. **部署到 /opt 再打包**  
   - 将 openclaw 与 openclaw-management 放到例如 `/opt/amyclaw/openclaw`、`/opt/amyclaw/openclaw-management`。  
   - 修改 systemd unit 中 `WorkingDirectory` 为 `/opt/amyclaw/...`。  
   - 用 eggs 打包时，/opt 会被 bind mount 进 liveroot，从而进入 ISO。

2. **打包后写脚本把应用打进 ISO**  
   - 先按上面用本地 snapshot_dir 打一版“系统+egg 默认内容”的 ISO。  
   - 再写脚本：挂载 ISO、解压/挂载 squashfs、向镜像内加入 `/mnt/disk/amyclaw`（或 `/opt/amyclaw`）所需目录、重新 mksquashfs、再打包 ISO。  
   - 文档与恢复流程中说明“应用由该脚本注入”。
