# 另一台机器恢复说明（AmyClaw Live 系统）

本说明用于在**另一台机器**上从 Penguins-eggs 生成的 Live ISO 恢复/安装系统，并让 OpenClaw 与管理前后端正常运行。

## 一、从 ISO 安装到硬盘

1. **获取 ISO**  
   从源机器的 `/mnt/backup` 拷贝生成的 ISO 文件（如 **v1.01**：`amyclaw_v1.01_20260316_0601.iso`）到 U 盘或网络可达位置。

2. **引导目标机**  
   目标机从 U 盘或 PXE 引导该 ISO，进入 Live 桌面。

3. **安装到磁盘**  
   - **图形安装**（若镜像包含 Calamares）：  
     在 Live 桌面运行「安装」或执行 `eggs calamares --install`，按向导将系统安装到目标机硬盘。  
   - **命令行安装**：  
     在 Live 下执行 `sudo eggs krill`，按提示完成安装。

4. **重启**  
   安装完成后从硬盘启动，进入已安装的系统。

## 二、首次进入系统后的配置

以下在**已安装好的系统**上执行（以 root 或带 sudo 的用户操作）。

### 2.1 系统用户与密码（若镜像未预置）

若打包时选择了**不包含系统 ID 和密码**，需自行创建用户并设置密码：

```bash
# 创建标准用户 amyclaw，密码与 root 一致（示例：amyclaw20260315）
sudo useradd -m -s /bin/bash amyclaw
echo "amyclaw:amyclaw20260315" | sudo chpasswd

# 可选：设置 root 密码
sudo passwd root
```

若镜像已预置用户 `amyclaw` 和 root 密码，可跳过或按需修改密码。

### 2.2 安装 Node.js 与依赖（若镜像内未带或版本不足）

OpenClaw 与管理端需要 Node.js ≥18：

```bash
# 示例：Ubuntu/Debian
sudo apt update
sudo apt install -y nodejs npm
# 若需 pnpm（OpenClaw 推荐）
sudo npm install -g pnpm
```

### 2.3 安装 OpenClaw 与管理端依赖

**v1.01 及之后**：应用已部署在 **/opt/amyclaw**（仅编译后运行文件，无源码）。若恢复的镜像是 v1.01，请使用：

```bash
export REPO=/opt/amyclaw
# 已包含 openclaw 与 openclaw-management；管理端若无 node_modules 需：
cd $REPO/openclaw-management && npm install 2>/dev/null || true
```

若为旧版或自建路径，将 `REPO` 改为实际路径（如 `/mnt/disk/amyclaw`），并在 openclaw 下执行 `pnpm install`、在 openclaw-management 下执行 `npm install`。

### 2.4 OpenClaw 状态目录与配置

镜像中**不包含** `/root/.openclaw`（已排除敏感），需在目标机上初始化（v1.01 使用 `REPO=/opt/amyclaw`）：

```bash
export OPENCLAW_STATE_DIR=/root/.openclaw
sudo mkdir -p "$OPENCLAW_STATE_DIR"
# 从量产模板复制预设配置（无密钥）
sudo cp -r $REPO/openclaw/production-template/state/* "$OPENCLAW_STATE_DIR/"
# 若有 .env.example，复制为 .env 并在管理页或本机填写 API Key
sudo cp "$OPENCLAW_STATE_DIR/.env.example" "$OPENCLAW_STATE_DIR/.env" 2>/dev/null || true
```

首次使用可在管理页（见下）生成或填写 Gateway Token，并填写飞书/企业微信/模型/向量等配置。

### 2.5 配置并启用 systemd 服务

**v1.01**：镜像内已包含指向 `/opt/amyclaw` 的 systemd 单元，安装后通常无需改路径。若需自定义路径，再改 unit 中的 `WorkingDirectory`。

安装并启用服务（状态目录为 `/root/.openclaw`）：

```bash
# v1.01：单元已在镜像 /etc/systemd/system/ 中，安装到硬盘后直接：
sudo systemctl daemon-reload
sudo systemctl enable openclaw-gateway openclaw-management
sudo systemctl start openclaw-gateway openclaw-management
```

### 2.6 验证

- **Gateway**：`curl -s http://127.0.0.1:18789` 或访问控制台/Web UI 的 Gateway 地址。
- **管理页**：浏览器打开 `http://<本机IP>:8080`，首次可生成或输入 Token，并保存飞书/企业微信/模型等配置。

## 三、可选：恢复路径与挂载

若目标机没有 `/mnt/disk/amyclaw`，可创建并挂载或拷贝：

```bash
sudo mkdir -p /mnt/disk/amyclaw
# 若从别处拷贝或挂载数据盘，请挂载到 /mnt/disk/amyclaw 后再执行 2.3–2.5
```

## 四、故障排查

| 现象 | 建议 |
|------|------|
| Gateway 或管理页起不来 | 检查 `systemctl status openclaw-gateway` / `openclaw-management`，确认 `WorkingDirectory`、`OPENCLAW_STATE_DIR` 正确，且已执行 `pnpm install` / `npm install`。 |
| 管理页提示 Token 错误 | 在管理页重新生成或填写 Token，并确认与 `openclaw.json` 中 `gateway.auth.token` 一致；状态目录需与 Gateway 一致（均为 `/root/.openclaw`）。 |
| 依赖缺失 | 在 `openclaw` 与 `openclaw-management` 目录下重新执行 `pnpm install` / `npm install`。 |

## 五、安全提醒

- 首次登录后请修改 root 与 amyclaw 密码（若为默认）。
- API Key、Token 等仅通过管理页或 `.env` 在本地配置，不要写进版本库或镜像。
- 若需远程访问管理页，建议配合防火墙与 HTTPS 反向代理使用。
