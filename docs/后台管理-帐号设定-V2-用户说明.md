# AmyClaw 后台管理 — 帐号与密码说明（用户版）

本文档面向**使用管理页面的用户**，说明如何登录管理后台、如何设置与修改「管理密码」以及如何修改本机系统登录密码。

---

## 一、管理后台是什么？

AmyClaw 管理后台是一个**网页设置页面**，用来：

- 填写或修改飞书、大模型、Gateway 等配置  
- 修改本机 **root** 和**普通用户**的登录密码  

打开方式：在浏览器地址栏输入 **`http://本机IP:8080`**（把「本机IP」换成您这台机器的实际 IP，例如 `http://192.168.1.100:8080`）。

---

## 二、两种不同的「密码」

请先分清两种完全不同的东西：

| 名称 | 用来做什么 | 在哪里设置/修改 |
|------|------------|----------------|
| **管理后台的登录凭证（Token）** | 用来**打开并登录**这个设置页面（8080 网页） | 首次打开页面时「设置 Token」；登录后在页面里「更改 Token」 |
| **本机系统密码（root / 用户密码）** | 用来**登录这台电脑**（SSH、插显示器键盘登录等） | 登录管理后台后，在「系统与用户密码」区块里修改 |

- 忘记「管理后台的 Token」→ 无法打开设置页，需要联系管理员查看或重置。  
- 忘记「本机系统密码」→ 无法登录这台电脑，但若还能打开管理页，可以在管理页里重新设置系统密码（需先能用 Token 登录管理页）。

---

## 三、首次打开管理页面：设置「管理密码」（Token）

1. 在浏览器打开 **`http://本机IP:8080`**。  
2. 若页面提示「未设置 Token」或要求先设置：  
   - **方式一**：点击 **「自动生成」**，再点击 **「确认并保存」**。  
     - 系统会生成一长串字符作为您的登录凭证；**请务必复制或截图保存**，以后每次登录管理页都要输入它。  
   - **方式二**：自己想一个长一点的、别人猜不到的字符串，输入到输入框，再点击 **「确认并保存」**。  
3. 保存成功后，您就相当于「设置了管理密码」；页面会进入设置主界面，之后可以填写飞书、大模型等配置。

**重要**：自动生成的 Token 只会在当时显示一次，请立即保存到安全的地方（如密码管理器或加密笔记），否则忘记后无法在页面上找回，只能联系管理员处理。

---

## 四、再次打开管理页面：用 Token 登录

1. 打开 **`http://本机IP:8080`**。  
2. 若页面要求输入 Token：  
   - 在输入框中粘贴或输入您**首次设置时保存**的那串 Token。  
   - 点击 **「确认并保存」**。  
3. 验证通过后会进入设置页；浏览器一般会记住登录状态，下次同机访问可能不需要再输。

若**忘记 Token**：无法在页面上自行找回，需联系管理员在服务器上查看或为您重新设置。

---

## 五、在管理页面里修改「本机系统密码」

登录管理后台后，页面上有 **「系统与用户密码」** 一块，可以修改：

- **root 密码**：本机系统管理员（root）的登录密码。  
- **用户密码**：本机普通用户（默认用户名是 **amyclaw**）的登录密码。

操作步骤（以 root 为例）：

1. 在「root」一栏输入**新密码**，并在「确认新密码」再输入一遍。  
2. 点击 **「修改 root 密码」**。  
3. 若提示成功，则本机 root 的登录密码已更新；之后用 SSH 或控制台登录时请用新密码。

**密码规则**（系统会检查）：

- 长度至少 **8 位**。  
- 必须同时包含：**大写字母**、**小写字母**、**数字**。  

例如：`MyPass123`、`Abc12345` 都符合；`12345678`（无字母）或 `abcdefgh`（无大写、无数字）会提示不符合规则。

修改「用户密码」时，规则相同，只是修改的是本机普通用户（如 amyclaw）的密码。

## 六、常见问题（用户侧）

**Q：页面打不开或一直转圈？**  
- 确认本机 IP 是否正确、是否已联网；确认端口是 8080；若在公司网络，确认防火墙或策略是否允许访问 8080。

**Q：提示「未授权」或 401？**  
- 表示您还没有用正确的 Token 登录。请在输入框输入您保存的 Token 并点击「确认并保存」。

**Q：忘记管理页的 Token 怎么办？**  
- 页面上无法找回，需要联系管理员在服务器上查看或重新设置 Token。

**Q：修改 root 或用户密码时提示错误？**  
- 检查新密码是否满足：至少 8 位，且同时包含大写字母、小写字母和数字；两次输入是否一致。

**Q：改完系统密码后，原来的 SSH 或控制台登录不上了？**  
- 您改的就是系统登录密码，改完后请用**新密码**登录；若新密码也忘了，需要管理员通过其他方式（如单用户模式）重置。

---

## 七、小结

- **管理后台**：浏览器打开 `http://本机IP:8080`，用 **Token** 登录，用于配置飞书/大模型等，以及修改本机 root 与用户密码。  
- **Token**：相当于管理页的「管理密码」，首次可自动生成或自己填写，**务必保存好**；忘记需找管理员。  
- **系统密码**：在管理页「系统与用户密码」里改，需满足 8 位 + 大小写字母 + 数字；改的是本机 root 和普通用户（如 amyclaw）的登录密码。

如有更多问题，请联系您的系统管理员或运维人员。
