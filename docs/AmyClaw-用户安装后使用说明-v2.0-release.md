# AmyClaw 用户说明 — 安装后设置与使用（v2.0-release）

> **定位**：面向**已安装到硬盘**的使用者（或现场工程师代操作）。**不包含** U 盘制作、母机打 ISO、量产排错（见 **`AmyClaw-量产与运维-v2.0-release.md`** 与 **`eggs-量产克隆与使用说明.md`**）。

---

## 1. 第一次开机要做什么？

1. **拔掉 U 盘**（若刚从量产 U 盘装完系统），从**硬盘**启动。  
2. 用 **网线** 或 **Wi‑Fi** 接入能上网的网络（默认 **DHCP** 自动获取 IP）。  
3. 在**同一局域网**的另一台电脑浏览器中打开管理页面（见下一节）。  

---

## 2. 打开管理设置页（8080）

### 2.1 查本机 IP（Ubuntu）

在**本机**打开终端（或 SSH 登录后），任选其一：

| 方式 | 命令 / 操作 |
|------|-------------|
| **快速** | `hostname -I`（多个地址时取连接管理电脑同一网段的那个） |
| **详细** | `ip -4 addr show scope global` 或 `ip -br a` |
| **看默认出口** | `ip route get 1.1.1.1`（输出中 `src` 后一般为当前用于对外通信的源地址） |
| **图形界面** | **设置 → 网络 → 有线 / Wi‑Fi → 详情**，查看 IPv4 地址。 |

也可从**路由器管理界面**查看设备名（如 **amyclaw**）对应的 IP。

### 2.2 浏览器访问

在浏览器地址栏输入：  

**`http://<本机IP>:8080`**  

例：`http://192.168.1.100:8080`  

---

## 3. 设置「管理密码」（Token）

1. 首次打开若提示设置 Token：  
   - 可使用 **「自动生成」** 并**立即复制保存**到安全位置；或  
   - 自行设置一串足够长的密码式字符串。  
2. **Token 即管理页登录凭证**，丢失后无法在本页自行找回，需管理员在服务器上处理。  

更细的步骤与修改 Token 说明见 **`后台管理-帐号设定-V2-用户说明.md`**（本文件不重复）。

---

## 4. 填写飞书与大模型（必做才能对话）

1. 登录管理页后，在对应页面填写：  
   - **飞书**：应用 **App ID**、**App Secret**（及机器人已加入群/单聊等，见飞书侧文档）。  
   - **大模型**：按页面提示填写 API Key（或提供商标识）。  
2. 点击 **保存**；若网关已运行，一般会触发热加载或提示重启（以页面提示为准）。  

**飞书开放平台**从注册账号到创建应用、机器人、权限与订阅：**见 `飞书-开放平台注册应用与机器人指南.md`**。

> **提醒**：**请勿**在多台 AmyClaw / 多套 OpenClaw 上**共用同一飞书应用帐号**做调试，否则易出现收不到消息、消息错乱，详见 **`AmyClaw-量产与运维-v2.0-release.md`** 第 5.4 节。

---

## 5. Control UI / 网关（18789，可选）

- 若需使用 OpenClaw **Gateway 自带界面**（非 8080 管理页）：浏览器访问 **`http://<本机IP>:18789`**。  
- 需使用与网关一致的 **Gateway Token**（通常与管理页/`.env` 中一致，具体以现场配置为准）。  
- 若仅局域网无法打开，可查看 **`18789-网关管理后台无法访问-排查.md`**。  

---

## 6. 修改本机登录密码（root / amyclaw）

在管理页的 **「系统与用户密码」** 中修改 **root** 与 **普通用户** 密码；规则见 **`后台管理-帐号设定-V2-用户说明.md`**。  

---

## 7. 常见问题

| 现象 | 建议 |
|------|------|
| 飞书发消息无回复 | 先确认飞书应用已启用、机器人已进群/会话；**管理端** 已保存；若仍不行，联系运维在服务器执行 OEM 自检脚本或查看 **`热加载与无回复排查.md`**。 |
| 大模型无回复 | 检查 Key 是否保存成功、模型名称是否可用；保存后稍等再试。 |
| 打不开 8080 | 检查 IP、防火墙、服务是否运行（`systemctl status openclaw-management` 等，需运维）。 |
